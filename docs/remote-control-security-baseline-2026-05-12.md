# 远程控制安全基线

更新时间：2026-05-12

## 1. 原则

- 真正执行始终在桌面端
- 服务端不替代桌面端执行敏感操作
- 移动端只发起控制意图，不直接成为资源执行主机

## 2. 高危操作清单

| 操作 | 风险等级 | 需要二次确认 | 需要审计记录 |
|------|---------|------------|------------|
| 关闭/重启 Agent Host | 高 | 是 | 是 |
| 强杀进程 | 高 | 是 | 是 |
| WSL 重置 | 高 | 是 | 是 |
| 文件删除 | 高 | 是 | 是 |
| 配置修改 | 中 | 是 | 是 |
| 会话终止 | 中 | 是 | 否 |
| 资源投递 | 中 | 否 | 是 |
| 查看上下文 | 低 | 否 | 否 |
| 查看状态 | 低 | 否 | 否 |

## 3. 二次确认流程

```text
移动端发起高危操作
  → 服务端路由到桌面端
  → 桌面端弹出确认 ActionCard
  → 等待远程确认/拒绝
  → 确认：执行操作并记录审计
  → 拒绝：不执行，返回拒绝原因
```

## 4. 资产加密边界

- 敏感文本、代码、密钥、日志默认按加密资产处理
- 同步时使用 E2EE（已有 FlowSync vault）
- 服务端只承担必要路由、存储和通知职责

## 5. 审计记录

- 每次高危操作记录：操作类型、时间、发起端、确认状态
- 审计日志存储在桌面端本地
- 可选择同步到服务端
