# 知识沉淀与跨会话资产流转设计

更新时间：2026-05-12

## 1. 可沉淀资产类型

| 资产类型 | 来源 | 目标 | 优先级 |
|---------|------|------|--------|
| 错误总结 | Session 输出、日志 | Knowledge Store | 高 |
| 配置片段 | Terminal 命令、设置页 | 模板库 | 高 |
| 命令片段 | 终端历史 | 命令模板 | 中 |
| 调试输出 | Agent 会话 | 诊断记录 | 中 |
| 代码片段 | 工作区文件 | 代码库 | 低 |

## 2. 跨会话流转路径

```text
Session A 产出
  → Artifact Harvester 提取
  → Knowledge Store 暂存
  → Session B 按需引用
  → 人工确认后注入
```

## 3. Storage Layer（知识存储）

建议分为两层：

- **Session 级暂存**：当前会话周期内的临时资产，会话关闭后可选择沉淀或丢弃
- **Host 级持久化**：跨会话的持久资产，存储在本地数据库或文件中

## 4. 与现有模块关系

- FlowSync：负责加密同步沉淀资产到其他设备
- FlowSnap：负责从沉淀资产生成可复用的模板
- Rules：决定哪些资产自动沉淀，哪些需要确认
- Host Recovery：在恢复时保留最近沉淀的资产
