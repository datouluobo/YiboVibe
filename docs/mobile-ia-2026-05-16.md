# YiboVibe 移动端 IA（简版）

更新时间：2026-05-16

## 1. 目标

把此前移动端 UI 稿收敛成一套可实现、可维护的最小信息架构。

移动端仍然只承担 `Remote Console` 角色，不复制桌面端全量页面，不扩展为独立执行宿主。

## 2. IA 结论

移动端收敛为 `1 个主链路 + 2 个前置/辅助层`：

```text
登录层
  -> 登录

主链路
  -> 远程控制台

辅助层
  -> Session 面板
  -> 事件/状态面板
```

这意味着首期不要把移动端拆成多级 Tab，也不要照搬桌面端 `FlowDeck / FlowRules / Settings` 的完整页面树。

## 3. 页面结构

### 3.1 登录

职责：

- 输入服务端地址
- 输入账号密码
- 录入当前手机设备名
- 完成鉴权并连接 Signal Hub

产出：

- `serverUrl`
- `token`
- `uid`
- `deviceId`

### 3.2 远程控制台

远程控制台是移动端唯一主页面，结构固定为：

```text
顶部状态区
  -> 当前连接状态
  -> 当前设备
  -> 当前 Session
  -> 当前 Session 状态

Session 快切区
  -> Session Chip 列表
  -> 新建 / 切换 / 关闭

主视图区
  -> 终端模式
  -> 对话模式

底部输入区
  -> 第一行：输入框 + 发送
  -> 第二行：工具条

辅助折叠区
  -> 事件日志
```

## 4. 信息分层

### 4.1 一级信息

用户每次打开移动端，优先看到：

- 是否连上桌面宿主
- 正在操作哪一个 Session
- 当前 Session 是否运行中
- 最近输出是否正常到达
- 我现在能不能继续输入

### 4.2 二级信息

仅在需要时展开：

- Session 生命周期动作
- 连接异常原因
- 事件日志
- 新建 Session 入口

### 4.3 暂不前置的信息

以下能力先不进入首屏 IA：

- 文件投喂
- FlowMind 面板
- 图片投递
- FlowProbe 控制面板
- 完整设置中心

这些能力后续应通过底部工具条入口或弹层接入，而不是先拆成独立主导航。

## 5. 交互约束

- 主链路保持单页，不依赖多层返回跳转。
- `Session` 切换优先通过横向快切或底部面板完成。
- 高危动作如 `Stop`、`Close` 必须二次确认。
- `终端模式` 与 `对话模式` 共享同一条 Session 事件流，只改变投影方式。
- 输入区固定在底部，不随输出滚动丢失。

## 6. 与当前代码的对应关系

- 登录层：`YiboVibe-Mobile/lib/pages/login_page.dart`
- 远程控制台：`YiboVibe-Mobile/lib/pages/console_page.dart`
- 鉴权与 Session 拉取：`YiboVibe-Mobile/lib/services/api_service.dart`
- Signal Hub 连接与消息收发：`YiboVibe-Mobile/lib/services/signal_client.dart`

本次先把 IA 落到两处：

1. 文档层：新增本说明，作为移动端页面结构的最小基线。
2. 代码层：把 Flutter 控制台页面整理成与 IA 一致的单页结构，明确顶部状态区、Session 快切、双模式主视图、双行底部输入区与事件区。
